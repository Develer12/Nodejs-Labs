Web Distributed Authoring and Versioning – расширение протокола HTTP/HTTPS, 
поддерживающее совместную работу по управление файлами на удаленных web-северах; 
применяется для  создания сетевой файловой системы;

2. Назначение WebDav. управление файлами на удаленных web-северах

основные функции WebDav.
o	Выполнение основных файловых операций над объектами на удаленном сервере;
o	Выполнение расширенных файловых операций (блокировки, поддержка версий);
o	Работа с любым типом объектов (не только файлы);
o	Поддержка метаданных (свойств) объектов;
•	Поддержка одновременной работы над объектами.

методов WebDav, расширяющих методы протокола HTTP.
•	PROPFIND — получение свойств объекта на сервере в формате XML. 
		Также можно получать структуру репозитория (дерево каталогов);
•	PROPPATCH — изменение свойств за одну транзакцию;
•	MKCOL — создать коллекцию объектов (каталог в случае доступа к файлам);
•	COPY — копирование из одного URI в другой;
•	MOVE — перемещение из одного URI в другой;
•	LOCK — поставить блокировку на объекте. 
		WebDAV поддерживает эксклюзивные и общие (shared) блокировки;
•	UNLOCK — снять блокировку с ресурса.
